# cc-web-control 部署使用文档

本文档面向“可直接上线/可直接运维”的使用场景，给出本机部署、隧道外网访问、鉴权、安全基线与故障排查。

## 1. 适用范围

- 项目路径：`/Volumes/work/workspace/cc-control`
- 目标：通过 Web 控制本机 tmux session（间接控制 Claude Code CLI）
- 默认运行方式：`node server.js` + 可选 `cloudflared tunnel`

## 2. 依赖要求

必需：

- Node.js（建议 >= 18）
- `tmux`
- `claude` CLI

可选：

- `cloudflared`（外网访问）

## 3. 部署模式

### 3.1 本机模式（推荐开发调试）

```bash
cd /Volumes/work/workspace/cc-control
CC_WEB_PROJECT_ROOTS="/Volumes/work/workspace" \
CC_WEB_NO_OPEN=1 \
CC_WEB_NO_ATTACH=1 \
node server.js --no-open --no-attach
```

访问：

- `http://127.0.0.1:7684`

### 3.2 受控外网模式（推荐生产/手机访问）

使用一键脚本：

```bash
cd /Volumes/work/workspace/cc-control
./scripts/restart_tunnel.sh
```

脚本行为：

- 生成新 token（`CC_WEB_AUTH_TOKEN`）
- 启动 tmux session：`cc-web-control`
  - window `server`：Node 服务
  - window `tunnel`：Cloudflare Quick Tunnel
- 输出 `URL` 和 `TOKEN`

手机访问：

1. 打开脚本输出的 `URL`
2. 进入 `/login` 输入 `TOKEN`
3. 登录后使用页面切换 session / project

## 4. 推荐安全基线

- 必开：`CC_WEB_AUTH_TOKEN`
- 默认监听：`CC_WEB_HOST=127.0.0.1`
- 避免路由器公网端口映射
- token 定期轮换（脚本每次重启自动更新）
- 优先使用“带 session 参数”的收藏 URL，减少误连：
  - `https://<domain>/?session=claude-xxx`

## 5. 关键环境变量

服务端：

- `CC_WEB_HOST`：监听地址（默认 `127.0.0.1`）
- `CC_WEB_PORT`：端口（默认 `7684`）
- `CC_WEB_SESSION`：默认 tmux session
- `CC_WEB_PROJECT_ROOTS`：允许项目根目录（逗号分隔）
- `CC_WEB_AUTH_TOKEN`：启用登录鉴权
- `CC_WEB_CLAUDE_CONTINUE`：`1` 时新启动 `claude` 使用 `-c`
- `CC_WEB_WEB_ONLY`：仅启动 Web 服务

隧道脚本（`scripts/restart_tunnel.sh`）：

- `CC_WEB_TMUX_SESSION`：tmux 会话名（默认 `cc-web-control`）
- `CC_WEB_PROXY_URL`：可选代理；默认空（不走代理）
- `CC_WEB_CLOUDFLARED_PROTOCOL`：默认 `http2`
- `CC_WEB_CLOUDFLARED_EDGE_IP_VERSION`：默认 `4`

## 6. 运维命令

查看服务会话：

```bash
tmux list-sessions
```

查看运行日志：

```bash
tmux attach -t cc-web-control
```

停止服务与隧道：

```bash
tmux kill-session -t cc-web-control
```

查看端口占用：

```bash
lsof -nP -iTCP:7684 -sTCP:LISTEN
```

## 7. 常见故障与处理

### 7.1 Cloudflare 1033

现象：

- `Cloudflare Tunnel error (1033)`

原因：

- 旧 quick tunnel 域名失效，或 tunnel 进程未连上边缘

处理：

1. 重新执行 `./scripts/restart_tunnel.sh` 获取新 URL
2. 检查 tunnel 日志：`/tmp/cc-web-control-tunnel.log`
3. 确认日志中有 `Registered tunnel connection ... protocol=http2`

### 7.2 页面显示未授权 / 反复重连

处理：

1. 用最新 token 重新登录 `/login`
2. 清理浏览器缓存后重开
3. 确认服务启动时确实设置了 `CC_WEB_AUTH_TOKEN`

### 7.3 session 记忆断层

处理建议：

1. 优先使用 URL 显式 session：`?session=...`
2. 开启 `CC_WEB_CLAUDE_CONTINUE=1`
3. 避免频繁销毁对应 tmux session

## 8. 升级与回滚

升级：

```bash
cd /Volumes/work/workspace/cc-control
git pull
npm install
./scripts/restart_tunnel.sh
```

回滚：

```bash
cd /Volumes/work/workspace/cc-control
git log --oneline -n 20
git checkout <commit>
npm install
./scripts/restart_tunnel.sh
```
