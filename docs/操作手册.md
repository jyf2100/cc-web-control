# cc-web-control 中文操作手册

> 本项目用于通过 Web 页面远程控制本机（或同一台机器上的）tmux session，从而间接控制 Claude Code CLI。**等价于远程操作一个终端会话**：请务必按本文的“安全模式/隧道模式”使用，避免无鉴权暴露到外网。

## 1. 你将得到什么

- 在浏览器中查看 tmux pane 输出（定时镜像 `tmux capture-pane`）
- 在浏览器中发送输入/按键到 tmux（`tmux send-keys`）
- 支持多会话（session 切换）与多项目启动（每个项目一个 tmux session）
- 可选：开启鉴权（token 登录）并通过 Cloudflare Tunnel 让手机外网访问

## 2. 环境准备

### 必需

- Node.js（建议 >= 18，仓库目前用 `node --test` 跑测试）
- `tmux`（建议 >= 3.0）
- Claude Code CLI（命令为 `claude`）

### 可选（外网访问/隧道）

- `cloudflared`（Cloudflare Tunnel 客户端）

### 可选（网络代理）

如果你的网络环境需要代理，本文统一使用：

- HTTP(S) 代理：`http://127.0.0.1:7890`

可用以下命令快速验证代理是否可用：

```bash
curl -I -x http://127.0.0.1:7890 --max-time 5 https://example.com
```

## 3. 安装依赖

进入项目目录：

```bash
cd /Volumes/work/workspace/cc-control
```

安装 Node 依赖：

```bash
npm install
```

如需通过代理安装：

```bash
HTTP_PROXY=http://127.0.0.1:7890 \
HTTPS_PROXY=http://127.0.0.1:7890 \
ALL_PROXY=http://127.0.0.1:7890 \
NO_PROXY=127.0.0.1,localhost \
npm install
```

## 4. 本机启动（仅本机访问）

最常用的启动方式（不自动打开浏览器、不 attach 到 tmux）：

```bash
cd /Volumes/work/workspace/cc-control
CC_WEB_PROJECT_ROOTS="/Volumes/work/workspace" node server.js --no-open --no-attach
```

浏览器打开：

- `http://127.0.0.1:7684`

### 4.1 只启动 Web（不创建/附加 tmux）

```bash
CC_WEB_WEB_ONLY=1 node server.js --no-open --no-attach
```

## 5. 页面使用说明（和 Claude Code 操作习惯对齐）

### 5.1 顶部栏：Session / Project

- **Session 下拉框**：切换当前 Web 连接对应的 tmux session
- **刷新**：刷新 session 列表与 project 列表
- **Project 下拉框 + 启动**：在某个项目目录创建新 session 并启动 `claude`
  - 需要先设置 `CC_WEB_PROJECT_ROOTS`，否则不会显示 Project

### 5.2 输入与按键行为

- **Enter**：发送输入
- **Tab**：转发给 tmux 触发补全（会先把 Web 输入同步到 tmux 的当前行）
- **/**：
  - 在页面任意位置按 `/` 会自动聚焦输入框并进入“命令面板”模式
  - 只输入 `/` 并回车发送时，默认只同步 `/` **不回车**（避免误执行）
- **↑/↓/Esc**：
  - 在 slash 命令面板（输入以 `/` 开头）或输入为空时，方向键/ESC 会作为按键转发到 tmux，用于选择/退出

> 说明：本项目基于 `capture-pane` 快照，不是完整终端仿真；但已尽量让 `/` 面板与 `Tab` 补全的“操作习惯”对齐。

## 6. Session 一致性（避免“记忆断层”）

“记忆断层”常见原因是：Web 端连接到了 **不同的 tmux session**。

本项目已经做了以下一致性策略：

- URL 显式指定 `?session=xxx` 时，**永不自动切换**
- URL 未指定时：
  - 优先从浏览器 `localStorage` 恢复上一次选择的 session
  - 若不存在，则回退到服务端默认 session（`CC_WEB_SESSION`，默认 `claude-web-session`）或列表中的合适 session
- 登录跳转会保留原始 URL（含 `?session=`），登录后会回到原页面

建议你在手机收藏夹里保存“带 session 的 URL”，例如：

```
https://<你的隧道域名>/?session=claude-evomap
```

## 7. 让“重建 session 后也继续最近对话”（claude -c）

即便 session 一致，如果 tmux session 被重建、需要重新启动 `claude`，Claude Code CLI 默认会开启新对话。

你可以开启：

- `CC_WEB_CLAUDE_CONTINUE=1`

效果：

- 当服务端需要新启动 `claude` 时，会使用 `claude -c/--continue` 在该目录继续最近一次对话，减少“对话断层”。

## 8. 安全模式（强烈推荐：token 登录）

开启鉴权：

```bash
export CC_WEB_AUTH_TOKEN="请设置为一个随机长字符串"
```

行为：

- 未登录访问 `/` 会跳转 `/login`
- 登录后设置 HttpOnly Cookie
- API 与 WebSocket 同样需要授权

## 9. 外网访问（手机 4G/5G）：安全隧道方案（推荐）

### 9.1 安装 cloudflared（可走代理）

```bash
HTTP_PROXY=http://127.0.0.1:7890 \
HTTPS_PROXY=http://127.0.0.1:7890 \
ALL_PROXY=http://127.0.0.1:7890 \
NO_PROXY=127.0.0.1,localhost \
brew install cloudflared
```

### 9.2 一键重启（自动打印 URL + 新 token）

脚本：

- `scripts/restart_tunnel.sh`

用法：

```bash
cd /Volumes/work/workspace/cc-control
./scripts/restart_tunnel.sh
```

脚本会：

- 生成新的 `CC_WEB_AUTH_TOKEN`
- 启动/重启一个 tmux session（默认名：`cc-web-control`）
  - window1：`server`（Node 服务）
  - window2：`tunnel`（cloudflared quick tunnel）
- 自动打印：
  - `URL: https://*.trycloudflare.com`
  - `TOKEN: <新 token>`

手机访问步骤：

1. 手机打开 URL
2. 进入 `/login` 输入 TOKEN
3. 进入主页面后选择 session / 项目即可

### 9.3 隧道/安装走代理

脚本默认代理为：

- `CC_WEB_PROXY_URL=http://127.0.0.1:7890`

可覆盖：

```bash
CC_WEB_PROXY_URL="http://127.0.0.1:7890" ./scripts/restart_tunnel.sh
```

## 10. 常用运维命令

查看 tmux session：

```bash
tmux list-sessions
```

进入隧道/服务日志：

```bash
tmux attach -t cc-web-control
```

停止服务与隧道：

```bash
tmux kill-session -t cc-web-control
```

## 11. 故障排查

### 11.1 端口被占用（EADDRINUSE）

说明：`7684` 已被其它进程占用。

排查：

```bash
lsof -nP -iTCP:7684 -sTCP:LISTEN
```

解决：

- 关闭占用进程或改端口：`CC_WEB_PORT=xxxx`

### 11.2 WebSocket 显示 Unauthorized / 页面反复重连

说明：你开启了 `CC_WEB_AUTH_TOKEN`，但未登录或 Cookie 失效。

解决：

- 打开 `/login` 重新输入 token
- 使用脚本时以最新输出的 `TOKEN` 为准

### 11.3 Project 下拉框不显示

说明：未设置允许扫描的根目录。

解决：

```bash
export CC_WEB_PROJECT_ROOTS="/Volumes/work/workspace"
```

### 11.4 session “missing”

说明：URL 或 localStorage 指向的 session 在 tmux 中不存在。

解决：

- 用页面顶部 Session 下拉框切换到存在的 session
- 或在命令行查看：`tmux list-sessions`

## 12. 安全建议（强提醒）

- **不要**在未开启 `CC_WEB_AUTH_TOKEN` 的情况下做外网隧道。
- 不要做“路由器端口映射到公网”（风险极高）。
- token 建议定期更换；不要发到群里/截图里。
- 该工具会镜像终端输出：请避免在被镜像的终端里输出敏感信息。
